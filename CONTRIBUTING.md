# Contributing to MeshSats

First off, thank you for considering contributing to MeshSats! It's people like you that make MeshSats such a great tool for the Bitcoin ecosystem and African financial inclusion.

## Welcome!
MeshSats is an open-source project focused on building offline-first Bitcoin infrastructure for low-connectivity environments. We welcome contributions from developers, technical writers, and domain experts in Bitcoin/Lightning infrastructure.

## How Can I Contribute?

### Reporting Bugs
*   Check the [issue tracker](https://github.com/bethelclement/MeshSats/issues) to see if the bug has already been reported.
*   If not, create a new issue. Include a clear title, a detailed description, steps to reproduce, and any relevant logs or screenshots.

### Suggesting Enhancements
*   We use GitHub issues for feature requests.
*   Clearly explain the use case and why this enhancement would be valuable for African merchant or agent networks.

### Pull Requests
1.  Fork the repo and create your branch from `main`.
2.  If you've added code that should be tested, add tests.
3.  If you've changed APIs, update the documentation.
4.  Ensure the test suite passes.
5.  Make sure your code lints.
6.  Issue that pull request!

## Style Guide
*   **Documentation**: We value clear, technical, and grounded documentation. Avoid hype and vague jargon.
*   **Code**: Follow the standard conventions for the respective language (Go, Rust, TypeScript, etc.).
*   **Commits**: Use descriptive commit messages.

## Community
Join us in building the future of offline Bitcoin payments!
